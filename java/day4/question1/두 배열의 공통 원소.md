### 🟢 문제 1 — 두 배열의 공통 원소 (워밍업)

**[백준 스타일]**

**제한**
- 시간: 1초 / 메모리: 256MB
- 1 ≤ N, M ≤ 100,000
- 1 ≤ 각 원소 ≤ 1,000,000

**문제**
두 배열 A, B가 주어진다. 두 배열 모두에 존재하는 원소를 **오름차순**으로 출력하라. 중복 원소는 한 번만 출력한다.

**입력 형식**
```
첫째 줄: N
둘째 줄: N개의 정수 (배열 A)
셋째 줄: M
넷째 줄: M개의 정수 (배열 B)
```

**출력 형식**
```
공통 원소를 오름차순으로 한 줄에 공백 구분 출력
공통 원소가 없으면 아무것도 출력하지 않음
```

**예제 입력 1**
```
5
1 2 3 4 5
4
3 5 7 9
```
**예제 출력 1**
```
3 5
```

**예제 입력 2**
```
3
10 20 30
3
40 50 60
```
**예제 출력 2**
```

```

> 💡 체크포인트: `ArrayList.contains()`로 풀면 O(N*M)이에요. N=100,000이면 100억 연산 → TLE. `HashSet`을 쓰면 O(1) 조회로 해결돼요. `TreeSet`을 쓰면 정렬까지 자동으로 해줘요.

### 🔴 문제 3 — 실시간 중앙값 (도전)

**[백준 스타일]**

**제한**
- 시간: 1초 / 메모리: 256MB
- 1 ≤ N ≤ 100,000
- 1 ≤ 각 수 ≤ 1,000,000

**문제**
수가 하나씩 들어올 때마다, 지금까지 들어온 수들의 **중앙값**을 출력하라. 짝수 개이면 작은 쪽 중앙값을 출력한다.

예: `[3]` → 3, `[3,1]` → 1, `[3,1,4]` → 3

**입력 형식**
```
첫째 줄: N
둘째 줄: N개의 정수
```

**출력 형식**
```
N개의 중앙값을 공백으로 구분해 한 줄 출력
```

**예제 입력 1**
```
5
3 1 4 1 5
```
**예제 출력 1**
```
3 1 3 1 3... 
```

잠깐, 직접 확인해보세요. 숫자 하나씩 추가하면서 중앙값이 뭔지 손으로 써보는 것도 풀이 템플릿 3단계예요.

> 💡 체크포인트: N개마다 정렬하면 O(N² log N) → TLE. `PriorityQueue` 두 개 (maxHeap + minHeap) 를 쓰면 O(N log N)으로 해결돼요. maxHeap은 중앙값 이하, minHeap은 중앙값 초과를 담아요.
